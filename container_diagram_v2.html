Vendor / Contractor
[Person]
Completes factory work items, submits handover documents, and asks natural-language questions.
Customer / Reviewer
[Person]
Reviews AI audit results, queries knowledge, and manages audit templates.
AI Document Audit & Knowledge Assistant System [Software System]
Frontend
[Container: JavaScript, React]
Conversational + audit UI: document upload, audit results, NL query, and template management.
Backend Service
[Container: Python, FastAPI]
REST API. Checks Entra ID role for each action: upload (Vendor/Contractor), review (Customer/Reviewer), view/query knowledge (any authenticated employee). No content-scope mapping.
Connector Layer
[Container: Python, Celery] (evolved from ETL Service)
Per-source connectors ingest content from SharePoint, Confluence, file servers, and databases, tagging it by source system.
AI Service
[Container: Python]
Routes intent (search / summarize / process-guidance / expert-lookup) and runs RAG over the unified index.
Expert Directory Service
[Container: Python]
Resolves subject-matter experts from document authorship metadata and org/HR profiles.
Document Storage
[Azure Blob Storage]
Stores uploaded handover documents and crawled files.
Database
[PostgreSQL]
Stores user data, audit records, results, and templates.
Search Index
[Azure AI Search]
Unified vector index: audit docs + multi-source knowledge + process content, tagged by source system.
Message Broker
[Redis]
Message queue for Celery task distribution.
Feedback & Learning Store
[PostgreSQL]
Reviewer corrections queued for curator flag-and-review; approved ranking/prompt config is read by AI Service.
Governance & Audit Log
[PostgreSQL]
Logs every Q&A/audit interaction with sources surfaced and permission context applied.
Azure Entra ID
[Software System]
Authentication and role/department identity for system users.
Enterprise Content Sources
[Software System]
SharePoint, Confluence, file servers, and internal databases — corporate knowledge and structured data.
Azure Content Understanding
[Software System]
Analyzes document content to extract text and structure.
Azure OpenAI Service
[Software System]
LLM capabilities for answers, summaries, and audit results.
HR / Org Directory
[Software System]
Employee profiles for expert identification.
Container Diagram v2 — AI Document Audit & Knowledge Assistant System
Gold outline = new/changed since v1. Same person actors as v1. Access is action-level only (upload / review / view), checked against Entra ID roles — no entitlement store, no content-scope tagging. Expert directory seeded from document metadata + HR directory. Reviewer feedback flows to a human-gated flag-and-review queue.
